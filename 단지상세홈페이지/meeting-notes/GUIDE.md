# 회의록 작성 가이드

> **적용 프레임워크: C²ARE**
> Context → Categories → Actions → Resolutions → Expansion
> "모든 대화는 맥락이고, 모든 맥락은 행동이 된다."

## 1. 파일 네이밍

```
YYYYMMDD_회의명.md
```

예시: `20260601_호강노노_신규기능_기획리뷰.md`

## 2. C²ARE 프레임워크 — 회의록 템플릿

```markdown
# [회의명]

**일시**: YYYY-MM-DD HH:MM  
**장소**: 온라인 / 오프라인 / 회의실명  
**참가자**: 이름(역할), 이름(역할), ...  
**작성자**: 에반  
**회의 유형**: 의사결정 / 정보공유 / 문제해결 / 브레인스토밍

---

## Summary

> (회의 전체를 3문장 이내로 압축. 이 회의가 왜 열렸고, 무엇이 결정됐으며, 다음 단계가 무엇인지를 포함.)

---

## 회의 배경 및 맥락

- **배경**: 이 회의가 열린 이유와 직전 상황
- **목표**: 이 회의에서 달성하려 했던 것
- **전제 조건**: 논의에 영향을 준 제약, 가정, 이전 결정 사항

---

## 주요 안건

> 카테고리는 회의 성격에 따라 조정. 각 항목은 "무엇이 논의됐는가"를 명시적으로 기록.

### [카테고리 1 — 예: 기능 정의]

- **논의 내용**: 어떤 의견이 오갔는가

- **미결 사항**: 결론에 이르지 못한 부분 (있을 경우)

## 결정 사항

| # | 결정 내용 | 결정 근거 | 결정자 |
|---|-----------|-----------|--------|
| 1 |           |           |        |
| 2 |           |           |        |

> 결정되지 않은 것은 이 표에 올리지 않는다. 조건부 결정은 조건을 명시한다.

---

## 액션 아이템

| # | 액션 내용 | 담당자 | 기한 | 우선순위 | 완료 기준 |
|---|-----------|--------|------|----------|-----------|
| 1 |           |        |      | 높음/중간/낮음 | |
| 2 |           |        |      |          |        |

> - **기한 없는 항목은 존재하지 않는다.** 모르면 "협의 후 확정"으로 명시.
> - 완료 기준은 "누가 봐도 완료 여부를 판단할 수 있는" 문장으로 작성.
> - 회의 내 모든 "~해보자", "~검토하자", "~확인하자" 발언은 액션 아이템으로 전환.

---

## 확장 고려사항

> 회의 내용을 바탕으로, 참석자들이 미처 다루지 못했지만 앞으로 영향을 줄 수 있는 사항.
> 2차 효과(Second-order effects), 숨은 리스크, 연관 의사결정 포인트를 포함.

### 추가로 고려할 사항

- **관련 이해관계자**: 이 결정에 영향을 받지만 회의에 없었던 사람/팀
- **잠재 리스크**: 현재 결정이 만들어낼 수 있는 문제
- **확인이 필요한 전제**: 논의에서 검증 없이 사용된 가정
- **연계 검토 항목**: 이 회의와 연결된 다른 의사결정 또는 프로젝트

---
```

## 3. 작성 원칙

### 문장 원칙
- **주어를 생략하지 않는다.** "검토한다" → "에반이 검토한다."
- **수동태를 피한다.** "논의됐다" → "팀이 논의했다."
- **추상어를 숫자/이름으로 바꾼다.** "빨리 진행" → "6월 5일까지 진행"
- **맥락 없이 단어만 쓰지 않는다.** "UX 이슈" → "모바일 상세페이지 스크롤 시 하단 버튼이 가려지는 UX 이슈"

### 맥락 포착 원칙
- 회의 중 지나가듯 나온 발언도 이후 의사결정에 영향을 줄 수 있다. **"~인 것 같다", "~하면 어떨까"류 발언도 기록**하되, 결정 사항과 구분한다.
- 말의 온도(강한 반대, 조건부 동의, 유보 등)도 가능한 한 기록한다.
- 회의 전 제공된 자료나 선결정 사항이 있다면 배경 섹션에 명시한다.

### 액션 아이템 전환 원칙
다음 표현이 등장하면 무조건 액션 아이템으로 전환한다.

| 회의 중 발언 패턴 | 전환 방식 |
|------------------|-----------|
| "~해보자" | 담당자 지정 + 기한 설정 |
| "~확인해줘" | 확인 주체 + 완료 기준 명시 |
| "~검토가 필요할 것 같아" | 검토자 + 검토 범위 + 기한 |
| "나중에 얘기하자" | 다음 회의 안건으로 등록 |
| "공유해줄게" | 공유 대상 + 시점 명시 |

## 4. Claude에게 회의록 요청하는 방법

아래 형식으로 제공하면 C²ARE 프레임워크에 따라 즉시 작성합니다.

```
회의명: 
일시: 
참가자: 이름(역할) 형식으로
회의 유형: 의사결정 / 정보공유 / 문제해결 / 브레인스토밍

[음성 메모 내용 또는 메모 붙여넣기]
```

> 음성 메모가 거칠거나 중복이 있어도 괜찮습니다.
> 맥락을 최대한 살려 정리하고, 액션 아이템과 확장 고려사항을 자동으로 도출합니다.
