## 1. 프로젝트 개요

> 분양 광고주(건설사·분양대행사)가 호갱노노 채널톡 등으로 유입된 고객을 **관심고객등록 전환**까지 끌고 가는 분양 랜딩페이지의 **표준 템플릿**을 만든다. 실제 운영 중인 분양 홍보 랜딩(detre2.urr.kr, 디에트르 더 리체Ⅱ)의 검증된 정보 구조를 가져오되, 호갱노노 브랜드 톤과 호갱노노 분양 상세식 정보 카드, 그리고 전환 중심 설계를 결합한다.

**한 줄 요약**: detre2 구조 기반 + 호갱노노 양식으로 재구성한, 관심고객등록 전환에 최적화된 분양 랜딩페이지 표준 템플릿(샘플 1종 + 기획서).

**사업 범위**:
- 포함:
  - 분양 랜딩 표준 섹션 구조 정의 (히어로 → 분양 요약 카드 → 입지 → 사업개요 → 프리미엄 → 세대안내 → 단지안내 → 관심고객등록 → 면책)
  - 호갱노노 브랜드 톤(컬러·타이포·정보 카드) 적용
  - 전환(관심고객등록) 폼 설계 및 표시광고법·개인정보 컴플라이언스 가이드
  - 데스크탑·모바일 반응형, 모바일 하단 고정 CTA
  - 단일 HTML 샘플 (외부 의존성 없음, 더블클릭 열람)
- 제외:
  - 실제 단지 이미지·평면도·조감도 등 디자인 에셋 제작 (현재 플레이스홀더)
  - 백엔드 폼 처리·DB/CRM 연동 (샘플은 프론트 시연만)
  - 분양가·세대수 등 실제 사업 데이터 확정 (입주자 모집공고 기준 별도)
  - 광고 매체 송출·트래킹 픽셀 세팅

## 2. 배경 및 목적

### 문제 인식

- **현재 상황**:
  - 분양 랜딩이 광고사·현장별로 제각각이라 정보 구조·전환 설계·법무 안정성의 편차가 크다.
  - 어워즈 프로젝트에서 "랜딩 일관성(채널톡 문구↔랜딩 연결)"과 "관심고객등록 전환"을 핵심 평가축으로 잡았으나, **기준이 되는 표준 랜딩 레퍼런스가 없다.**
  - 기존 분양 홍보 랜딩(detre2 등)은 정보는 충실하나 구형 마크업·과장 카피("소유할 마지막 기회") 등 표시광고법 리스크가 섞여 있다.
- **발생하는 문제**: 클릭은 발생해도 관심고객등록까지의 전환 설계가 약하고, 광고사에 "이렇게 만들라"고 제시할 표준 산출물이 없다.
- **해결하지 않으면**: 어워즈의 "Best Landing"·"랜딩 개선 가이드 자산화"가 추상적 평가에 그치고, 분양 랜딩 품질이 계속 광고사 역량에 좌우된다.

### 해결 방향

> 실제 검증된 분양 랜딩(detre2)의 정보 구조를 표준화하고, 여기에 **① 호갱노노 양식(브랜드 톤·분양 요약 카드) ② 전환 중심 구조 ③ 컴플라이언스 가이드**를 얹어 광고사가 그대로 따라 쓸 수 있는 표준 템플릿으로 만든다.

#### 이 프로젝트의 본질

| 겉으로 보이는 목적 | 실제 목적 |
|---|---|
| 분양 홈페이지 하나 제작 | 어워즈 "Best Landing"의 기준이 되는 표준 레퍼런스 확보 |
| detre2 따라 만들기 | 검증 구조 + 호갱노노 양식 + 컴플라이언스로 업그레이드 |
| 예쁜 랜딩 | 관심고객등록 전환률을 끌어올리는 전환 설계 |
| 일회성 샘플 | 광고사 배포용 "랜딩 개선 가이드" 자산화 |

**필수 포함 고지(샘플 하단 고정)**: 홍보물 참고용 안내, 이미지·평면 연출 고지, 분양가·세대수·일정 변경 가능 및 입주자 모집공고 기준 안내.

**개인정보**: 필수 수집은 이름·연락처로 최소화. 생년월일·주소 등은 선택. 분양대행사 제3자 제공 동의를 명시하고, 이용목적·보유기간을 표기한다.

# 부록 D. 사용 방법 (샘플)

- 파일: `직방-호갱노노-분양홈페이지-템플릿-샘플/index.html`
- 실행: 파일 더블클릭 → 브라우저에서 바로 열림 (외부 라이브러리·서버 불필요)
- 커스터마이즈: 상단 `:root` CSS 변수로 컬러 일괄 변경, 섹션별 텍스트/플레이스홀더 교체
- 이미지: 회색 점선 박스(`.ph`)에 실제 조감도·평면도·배치도 이미지를 넣어 교체
- 폼: 현재는 프론트 시연(alert)만 동작. 운영 시 분양대행사 CRM/호갱노노 관심고객 DB로 연동 필요(미결 #2)
