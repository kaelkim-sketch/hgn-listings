## 1. 프로젝트 개요

> 채널톡 이미지 템플릿을 **목적·현장 유형·이벤트별로 라이브러리화**하고, 월별 메인 테마로 선제 운영해 광고사 제작 품질을 표준화하고 채널톡 전환 성과를 개선하는 운영형 프로젝트.

**한 줄 요약**: 현장마다 매번 새로 만들던 채널톡 시안을, 재사용 가능한 템플릿 라이브러리 + 월별 테마 운영 체계로 전환한다.

**사업 범위**:
- 포함:
  - 채널톡 이미지 템플릿 분류 체계 설계 및 핵심 세트 제작
  - 월별 메인 테마 기반 선제 기획·배포 운영
  - 광고사 배포용 템플릿/카피 가이드 + 제출 표준화
  - 우수 템플릿 라이브러리 축적 및 성과 리뷰
- 제외:
  - 경품 중심·CTR 단독 평가형 이벤트 운영
  - 모든 분류(120여 중분류)를 전수 제작하는 방식 — **핵심 세트 우선**
  - 랜딩/상품(앱푸시·단지마커) 자체 기획 (연계만)

## 2. 배경 및 목적

### 문제 인식

채널톡은 분양광고에서 유저와의 직접 접점이지만, 현재 운영에 아래 구조적 문제가 있다.

| 문제 | 설명 |
|---|---|
| 시안 품질 편차 | 광고사별 이미지·카피·CTA 완성도 차이가 큼 |
| 반복 제작 비효율 | 현장마다 유사 시안을 매번 새로 제작 |
| 메시지 구조 불일치 | 현장별 카피·정보 배열·CTA 기준 상이 |
| 전환 목적 불명확 | 관심등록형/상담형/청약형 구분 없이 혼용 |
| 이벤트 남발 위험 | 클릭만 유도하고 실제 전환 품질 저하 |
| 랜딩 연결 약함 | 채널톡 문구와 랜딩 불일치로 이탈 |

### 해결 방향

> 채널톡 템플릿을 "현장별 일회성 시안"이 아니라, **목적·현장 유형·이벤트 목적에 따라 반복 활용하는 광고 운영 자산**으로 설계한다. 분류는 넓게 갖추되 **매월 실제 운영은 메인 1 + 보조 2~3 + 이벤트 1~2 (총 5~7종)으로 통제**한다.

### 운영 철학 (원문에서 채택)

- **분류는 넓게, 운영은 집중**: 라이브러리는 다양하게, 월 노출은 선별.
- **이벤트형은 보조 장치**: 채널톡의 중심이 되어선 안 됨. 정보형 60 / 전환형 25 / 이벤트형 15 비중(※ 가설 — 4번·검토 의견 참조).
- **이벤트 = 전환 목적과 결합**: 단순 경품이 아니라 관심등록·상담·방문예약과 연결.

## 3. 목표 및 KPI

### 목표 (Objectives)

- 목표 1: 광고사별 시안·카피 품질 편차를 줄이고 제작 효율(시간·비용)을 높인다.
- 목표 2: 전환 목적(관심등록·상담·청약·잔여세대)별로 템플릿을 구조화해 전환 성과를 개선한다.
- 목표 3: 성과 검증된 우수 템플릿을 라이브러리로 자산화한다.

### KPI 측정 프레임 (퍼널 단계별)

> ⚠️ 절대 목표치는 baseline 측정 후 확정(미결 #4). 아래는 측정 틀 + 목표 설정 방식이다.
> **baseline** = 운영 직전 3개월 채널톡 집행 데이터 평균. **측정 주기** = 월별 측정 + 분기 종합 리뷰.

| 퍼널 | 핵심 지표 | 정의·측정 | 1차 목표 (가이드·가설) |
|---|---|---|---|
| 공통(품질) | 템플릿 채택률 | 집행 시안 중 템플릿 기반 비중 | 3개월 내 50%+ |
| 인지 | 열람률 | 발송 대비 열람 | baseline 유지·개선 |
| 관심 | 실질 CTR | 클릭 / 발송 | baseline +5~15% |
| 관심 | 관심고객등록률 | 클릭 → 등록 전환 | baseline +10~20% |
| 전환 | 상담신청률 | 등록·클릭 → 상담 | baseline +10% |
| 전환 | 방문예약률 | 견본주택 방문 전환 | baseline 대비 측정 |
| 재참여 | 리타겟 재유입률 | 이탈 후 재방문 | baseline 대비 측정 |
| 품질 | 랜딩 이탈률 | 클릭 후 랜딩 이탈 | baseline 감소 |

> 1차 목표의 % 밴드는 **설정 가이드(가설)**이며, baseline 확정 후 팀 합의로 수치를 고정한다. 임의 수치를 목표로 확정하지 않는다.

### 운영 가설 검증 — 정보 60 / 전환 25 / 이벤트 15

이 비중은 **근거 없는 출발 가설**이다. 운영 2~3개월간 비중을 ±10%p 변주해 실질 CTR·등록률·**피로도(수신거부율)** 기준으로 검증한 뒤 고정한다(미결 #5).

## 검토 의견 — 논점 및 개선 제안

> 기획안을 그대로 수용하지 않고 구조적으로 검토한 결과. **방향성(라이브러리화·월별 테마·이벤트 통제)은 타당**하나, 분류 설계와 실행 전제에 다음 논점이 있다.

### 잘 잡은 점
- 라이브러리화 + 월별 테마라는 **운영 자산화 관점**이 명확하다.
- "이벤트형은 보조 장치"라는 통제 원칙과 60/25/15 비중 의도가 합리적이다.
- 경품·개인정보·과장 문구 등 **법무/브랜드 리스크를 선제 인지**했다.

### 짚어야 할 논점

1. **분류 체계가 MECE하지 않다 (중복 과다).**
   - 9.지역·생활권형 ↔ 10.상품성 강조형: `역세권형·학세권형·숲세권형`이 양쪽에 **그대로 중복**.
   - 1.AI 상담사형 ↔ 2.분양상담사 추천형: '추천' 기능 중복.
   - 3.관심고객등록형 ↔ 7.이벤트형(관심고객등록 이벤트형): 등록 유도 중복.
   - 4.신규현장 런칭형 ↔ 14.브랜드·프리미엄형: 브랜드 신규 중복.
   - → **15개 대분류를 통합하고, 중복 중분류를 제거**해야 분류가 운영 가능해진다.

2. **대분류가 서로 다른 차원을 한 줄에 섞었다.**
   - 목적/퍼널(등록·청약·미분양) · 소재 앵글(지역·상품성·브랜드·타겟) · 포맷(AI·설문·이벤트) · 타이밍(시즌·리타겟팅)이 같은 레벨에 나열됨.
   - → **3축 매트릭스로 재설계 권장**: `① 목적(퍼널 단계) × ② 소재 앵글 × ③ 포맷`. 그러면 템플릿 = "관심등록 × 지역 × AI상담" 식으로 조합 생성되어, 15개 대분류가 ~3축 8~9 요소로 줄고 조합은 더 풍부해진다.

3. **120여 중분류는 라이브러리 스펙이지 실행 계획이 아니다.**
   - 문서도 월 5~7종만 제작한다고 명시. 120개를 다 만들면 대부분 사장된다.
   - → **성과·빈도 높은 핵심 세트 20~30종을 먼저 정의**하고, 성과 기반으로 확장. (미결 #6)

4. **월별 테마 캘린더와 현장 청약 일정이 충돌한다.**
   - "5월=청약 테마"는 콘텐츠 마케팅 캘린더 가정. 그러나 실제 분양 캠페인의 청약·오픈은 **현장마다 시점이 다르다**.
   - → 레이어를 분리: **월별 테마 = 직방·호갱노노 공통 발신/라이브러리 기준**, **광고사 현장 캠페인 = 현장 일정 우선**. 둘을 동일 축으로 두면 운영이 어긋난다.

5. **KPI에 목표·baseline·측정 주기가 없고, 60/25/15는 근거 없는 가설이다.**
   - → baseline 측정 후 목표 설정. 60/25/15는 **가설로 명시**하고 운영 2~3개월 A/B로 검증. (미결 #4·#5)

6. **어워즈 프로젝트와 역할·변별력이 충돌할 수 있다.**
   - 어워즈는 광고사 '시안 완성도(30%)'를 평가하는데, 본 프로젝트가 **공통 템플릿을 제공하면 시안 편차가 줄어** 어워즈의 변별력이 약해진다. 또 어워즈의 '신규 템플릿 사용량' KPI = 본 프로젝트 템플릿.
   - → 두 프로젝트의 **역할 분담**(템플릿=기본 품질 상향 / 어워즈=템플릿 위 응용·성과 경쟁)과 평가 기준 정합성을 함께 정리. (미결 #8)

7. **실행 리소스·주체가 정의되지 않았다.**
   - 월 5~7종 × 12개월(연 60~84종) + 라이브러리 유지 = 상시 디자인/기획 부하. 제작 주체·캐파·데드라인 미정이면 계획이 작동하지 않는다. (미결 #1·#2)

8. **법무/개인정보/경품은 '검토 필요' 상태로 남아 있다.**
   - 인지는 했으나 미완. **검토 완료 전 경품·고가 이벤트 출시는 보류**가 안전하다. (미결 #7)

> **반영 현황**: 논점 1·2·3 → 부록 A-2 통합 분류 체계·핵심 세트 25종. 5(KPI·비중) → 섹션 3 측정 프레임·검증 계획. 6(어워즈 변별력) → '어워즈 정합성' 섹션. 4(레이어 분리)·7(리소스)·8(법무)은 미결사항에 제안/이월.

## 어워즈 프로젝트와의 역할 정합성

> 본 프로젝트(템플릿)와 [분양광고 크리에이티브 어워즈](./직방-호갱노노-분양광고-크리에이티브-어워즈.md)는 대상·KPI가 겹친다(검토 의견 #6). 역할을 분리하고 환류 구조로 연결한다.

### 역할 분담 — "바닥 vs 천장"

| 구분 | 템플릿 프로젝트 | 어워즈 |
|---|---|---|
| 목적 | 기본 품질의 **바닥**을 올림 (편차 축소) | 그 위의 **천장**을 봄 (응용·성과 경쟁) |
| 대상 | 모든 광고사 (상시 제공) | 참여 광고사 (기간 한정) |
| 산출 | 템플릿 라이브러리·가이드 | 우수 사례·수상 |

### 변별력 충돌 해소

- **우려**: 공통 템플릿이 시안 편차를 없애 어워즈 '시안 완성도(30%)' 변별력이 약해진다.
- **해소**: 어워즈의 '시안 완성도'를 *"템플릿 단순 사용"*이 아니라 *"현장 맞춤 응용 + 카피·타겟 차별화"*로 평가 기준 재정의. 템플릿은 **출발선**, 어워즈는 **그 위의 차별화**를 평가 (어워즈 기획팀과 협의 — 미결 #8).

### 환류 루프 (선순환)

```text
템플릿 라이브러리·가이드 제공
→ 광고사 캠페인 집행 (어워즈 참여)
→ 어워즈 성과 평가·수상
→ 우수작을 신규 템플릿으로 환류
```

### 지표·일정 정합

- **지표 일원화**: 어워즈 '신규 템플릿 사용량' = 본 프로젝트 '템플릿 채택률'. 한 곳에서 측정·공유.
- **일정 분리**: 어워즈 접수(6/11~7/31)와 템플릿 운영 시작(7월 제안)이 겹친다. 템플릿은 어워즈와 **별개 트랙**으로 운영하고, 어워즈 종료(8월) 후 **우수작 환류**를 정식 단계로 둔다.
